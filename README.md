# CO2 Station

Настольная станция мониторинга качества воздуха на ESP32-C3 с цветным
дисплеем, сенсором Sensirion SCD41 (CO2, температура, влажность), сенсорной
кнопкой и публикацией показаний по MQTT.

Подробная история проекта и обоснование выбора компонентов — в
[WHATISIT.md](WHATISIT.md).

## Возможности

- Измерение CO2, температуры и относительной влажности сенсором SCD41.
- Плавная анимированная индикация показаний на цветном дисплее с
  собственным шрифтом (через библиотеку
  [`jef-sure/dgx`](https://components.espressif.com/components/jef-sure/dgx/)).
- Публикация измерений по MQTT для интеграции в умный дом.
- Настройка Wi-Fi по BLE через приложение Espressif (с показом QR-кода на
  экране).
- Локальный Web-сервер для настройки MQTT, имени устройства и
  эмпирических поправок к показаниям сенсора.
- Сенсорная кнопка (TTP223) для управления дисплеем и режимами работы.

## Управление сенсорной кнопкой

| Действие              | Результат                                                                       |
| --------------------- | ------------------------------------------------------------------------------- |
| Короткое касание      | Включить / выключить дисплей.                                                   |
| Двойное касание       | Запустить локальный Web-сервер со страницей настроек (если Wi-Fi настроен).     |
| Удержание ~5 секунд   | Перейти в режим BLE-настройки Wi-Fi, показать QR-код для приложения Espressif. |

Если за 60 секунд новые настройки Wi-Fi не получены — устройство
возвращается к обычной работе с прежней конфигурацией. После успешной
настройки новой сети устройство перезагружается.

## Настраиваемые параметры (Web UI)

- Адрес MQTT-брокера.
- Имя устройства.
- Локальное имя хоста (mDNS).
- Эмпирические поправки к показаниям сенсора:
  - смещение для CO2;
  - смещение для температуры;
  - смещение для влажности.

Поправки применяются к текущим измерениям; сетевые параметры сохраняются
в NVS.

> Замечание о поправках. Это не полноценная калибровка, а эмпирическая
> коррекция по одной опорной точке: задаётся только смещение к текущим
> измерениям.

## Сборка

Требуется ESP-IDF **5.2** или новее. Целевая платформа — **ESP32-C3**.
Скрипт `misc/generate_tzones.py` берёт данные часовых поясов из системной
`zoneinfo`, а при её отсутствии (например, в native Windows) — из Python-пакета
`tzdata`.

Установите Python-зависимости в уже активированное окружение ESP-IDF:

```sh
python -m pip install -r requirements.txt
```

В Windows проще всего выполнить эту команду из `ESP-IDF PowerShell` или
`ESP-IDF CMD`, в Linux/macOS — после активации через `. $IDF_PATH/export.sh`.

```sh
. $IDF_PATH/export.sh
idf.py set-target esp32c3
idf.py build
idf.py -p /dev/ttyUSB0 flash monitor
```

Зависимые компоненты ставятся автоматически менеджером компонентов IDF из
[main/idf_component.yml](main/idf_component.yml). При желании их можно
добавить вручную:

```sh
idf.py add-dependency jef-sure/dgx
idf.py add-dependency jef-sure/scd4x
idf.py add-dependency espressif/qrcode
```

## Аппаратная часть

- **MCU:** ESP32-C3 (плата разработчика).
- **Сенсор:** Sensirion SCD41 (I²C).
- **Дисплей:** цветной TFT, подключение через драйверы `dgx`.
- **Кнопка:** ёмкостный сенсор TTP223 (бесконтактное касание).
- **Корпус:** 3D-печать, модель собрана во FreeCAD (см. WHATISIT).

## Web server 

### Особенности формирования страницы настроек

HTML-шаблон страницы настроек нарезается во время сборки скриптом
`misc/split_to_template.py` на массив константных фрагментов, между
которыми сервер вставляет значения из текущих настроек — без шаблонизатора
в рантайме.
